# ⚡ Оптимизация CPU/GPU для Hand Gesture 3D Control

## 🎯 Конфигурация системы

### Ваша конфигурация:
- **CPU:** Мощный процессор (основная вычислительная мощность)
- **GPU:** Встроенная графика (менее производительная)

### Оптимальная стратегия:
✅ **MediaPipe Hands → CPU** (использует мощь вашего процессора)  
✅ **OpenGL рендеринг → GPU** (даже встроенная графика хороша для 3D)  
✅ **Многопоточность → CPU** (параллельная обработка на нескольких ядрах)

### 4. Настройки производительности

**Новые параметры в `settings.json`:**
```json
{
  "performance": {
    "fps_limit": 60,
    "render_quality": "high",
    "use_vsync": true,
    "optimize_hand_tracking": true,
    "cpu_threads": 4,
    "use_cpu_for_tracking": true,
    "use_gpu_for_rendering": true,
    "frame_buffer_size": 3
  }
}
```

**Описание параметров:**
- `cpu_threads` - количество потоков CPU (4 для вашего процессора)
- `use_cpu_for_tracking` - использовать CPU для MediaPipe
- `use_gpu_for_rendering` - использовать GPU для OpenGL
- `frame_buffer_size` - размер буфера кадров для плавности

## 🔍 Технические детали

### Почему CPU лучше для MediaPipe?

1. **MediaPipe - это нейросеть**
   - Требует много математических вычислений
   - CPU лучше справляется с последовательными операциями
   - Ваш мощный CPU быстрее встроенной GPU

2. **Встроенная GPU ограничена**
   - Мало видеопамяти
   - Низкая пропускная способность
   - Не предназначена для ML задач

3. **Разделение задач**
   - CPU: обработка видео, распознавание жестов
   - GPU: только 3D рендеринг
   - Каждая часть делает то, что умеет лучше

---

### Как работает многопоточность?

```
Поток 1: Чтение кадра с камеры
Поток 2: Обработка MediaPipe (рука 1)
Поток 3: Обработка MediaPipe (рука 2)
Поток 4: Вычисление жестов и нормализация
     ↓
Главный поток: Обновление 3D объекта
```

**Преимущества:**
- Параллельная обработка
- Нет блокировок
- Максимальная загрузка CPU
- Плавная работа

## 🐛 Решение проблем

### Проблема: FPS низкий (<30)

**Решение:**
1. Уменьшите `cpu_threads` до 2
2. Измените `render_quality` на "medium"
3. Отключите `use_vsync`

### Проблема: CPU загружен на 100%

**Решение:**
1. Уменьшите `cpu_threads`
2. Закройте другие программы
3. Проверьте температуру CPU

### Проблема: Задержка распознавания

**Решение:**
1. Увеличьте `frame_buffer_size` до 5
2. Уменьшите разрешение камеры в `hand_tracker.py`
3. Используйте `model_complexity=0` вместо 1
